## 二、模块职责与数据流速查

### 2.1 核心模块一句话总结

| 包 | 职责 | 关键类 |
|----|------|--------|
| `pub` | 基础字典：仓库/库区/库位/货主/货品 | `CommonController`, `PubPos` |
| `in` | 入库业务流程（到货→上架） | `ArrivalNoticeServiceImpl` |
| `io.base` | 库存查询/冻结/流水 | `Inventory`, `ETransType` |
| `io.in` | 入库库存操作 | `InventoryInServiceImpl`, `InventoryBaseService` |
| `io.out` | 出库库存操作 | `InventoryOutServiceImpl` |
| `io.bill` | 单据库存操作 | 继承 `InventoryBaseService` |
| `out` | 出库业务流程（接单→发货） | `PickingOrderServiceImpl` |
| `bill` | 盘点/移库/损溢等调整类单据 | `BillLossSurplusServiceImpl` |

### 2.2 全局数据流图

```

        ↓
```

## 五、常用排查 SQL

### 5.1 库存相关

```sql
-- 查某库位的所有库存（含数量详情）
SELECT goods_code, batch_no,
       inventory_quantity, available_quantity, locked_quantity,
       inbound_date, expiry_date
FROM wms_inventory
WHERE pos_code = '你的库位编码'
  AND inventory_quantity > 0
ORDER BY goods_code, batch_no;

-- 验证库存等式（应该返回 0 行）
SELECT COUNT(*) FROM wms_inventory
WHERE ABS(inventory_quantity - available_quantity - locked_quantity) > 0.001;

-- 查某货品的库存分布
SELECT pos_code, batch_no, inventory_quantity, available_quantity
FROM wms_inventory
WHERE goods_code = '货品编码'
  AND inventory_quantity > 0
ORDER BY pos_code;

-- 查某库存的完整流水历史
SELECT
    trans_type,
    CASE trans_type
        WHEN 1 THEN '收货入库' WHEN 2 THEN '撤销收货'
        WHEN 3 THEN '调整入库' WHEN 4 THEN '调整出库'
        WHEN 5 THEN '上架移入' WHEN 6 THEN '上架移出'
        WHEN 9 THEN '移动入' WHEN 10 THEN '移动出'
        WHEN 12 THEN '拣货移出' WHEN 13 THEN '出库发货'
        WHEN 14 THEN '报损出库' WHEN 15 THEN '报溢入库'
        ELSE CONCAT('type=', trans_type)
    END as type_name,
    trans_quantity, trans_doc_no, operation_time
FROM wms_inventory_trans_detail
WHERE inventory_id = 你的库存ID
ORDER BY id;

-- 查某库存的当前锁定情况
SELECT lock_type,
       CASE lock_type
           WHEN 1 THEN '上架锁定' WHEN 2 THEN '出库锁定'
           WHEN 3 THEN '移动锁定' WHEN 4 THEN '移库锁定'
           WHEN 5 THEN '养护锁定'
       END as lock_name,
       lock_doc_no, locked_quantity, lock_time
FROM wms_inventory_lock
WHERE inventory_id = 你的库存ID;
```

### 5.2 入库相关

```sql
-- 查一张到货通知单的完整状态
SELECT
    id, arrival_no, order_type, order_status, confirm_status,
    inspect_accept_status, qualify_result,
    planned_quantity, received_quantity, inspect_accept_quantity
FROM wms_arrival_notice WHERE id = 你的ID;

-- 查该单生成的所有相关数据
-- 收货单
SELECT id, receipt_no, order_status FROM wms_receipt
WHERE arrival_notice_id = 你的ID;
-- 质检单
SELECT id, inspection_no, status, qualify_result FROM wms_quality_inspection
WHERE arrival_notice_id = 你的ID;
-- 上架单
SELECT id, putaway_no, putaway_type, status FROM wms_putaway
WHERE arrival_notice_id = 你的ID;
-- 任务
SELECT task_no, task_type, status FROM wms_task
WHERE business_doc_no IN (
    SELECT arrival_no FROM wms_arrival_notice WHERE id = 你的ID
);
```

### 5.3 出库相关

```sql
-- 查出库单完整状态
SELECT outbound_order_no, order_status, order_type,
       order_quantity, allocated_quantity, picked_quantity,
       packed_quantity, shipped_quantity, intercepted_quantity
FROM wms_outbound_order WHERE id = 你的ID;

-- 查该出库单的拣货明细（了解从哪个库位拣的货）
SELECT pd.pos_code, pd.goods_code, pd.batch_no,
       pd.planned_picking_quantity, pd.picked_quantity, pd.status
FROM wms_picking_detail pd
WHERE pd.picking_order_no IN (
    SELECT picking_order_no FROM wms_picking_order
    WHERE outbound_order_no = '你的出库单号'
);

-- 查出库单对应的库存变动流水
SELECT trans_type, trans_quantity, trans_doc_no, operation_time
FROM wms_inventory_trans_detail
WHERE trans_doc_no = '你的出库单号'
ORDER BY operation_time;
```

### 5.4 问题排查

```sql
-- 查库存出现负数的情况（不正常）
SELECT * FROM wms_inventory
WHERE inventory_quantity < 0 OR available_quantity < 0;

-- 查长时间未完成的拣货单（可能是任务卡住了）
SELECT picking_order_no, status, create_time
FROM wms_picking_order
WHERE status IN (1, 2)   -- 待拣选或拣货中
  AND create_time < NOW() - INTERVAL 7 DAY  -- 超过 7 天未完成
ORDER BY create_time;

-- 查未完成的上架任务
SELECT task_no, task_type, status, business_doc_no, create_time
FROM wms_task
WHERE task_type = 2   -- 2=上架任务
  AND status IN (1, 2) -- 新建或执行中
ORDER BY create_time;
```

## 六、核心方法调用速查

### 6.1 入库核心链路

```
【接单】
ArrivalNoticeController.add()
  → ArrivalNoticeServiceImpl.add()
  → wms_arrival_notice 表（新建）

【确认】
ArrivalNoticeController.confirm()
  → ArrivalNoticeServiceImpl.confirm()
  → wms_task 表（生成收货任务）
  → wms_arrival_notice（confirm_status=2）

【验收+库存入账】
ArrivalNoticeController.inspect()
  → ArrivalNoticeServiceImpl.inspectAccept()
  → ReceiptDetailServiceImpl（更新收货明细）
  → generatePutawayOrder()
      → wms_putaway + wms_putaway_detail + wms_task（上架任务）
      → InventoryInServiceImpl.arrival()
          → InventoryBaseService.createInventory()  [wms_inventory 新记录]
          → InventoryBaseService.createInventoryTransDetail(ARRIVAL_IN)
          → InventoryBaseService.updateInventoryByLock(PUTAWAY)
  → wms_arrival_notice（order_status=3, inspect_accept_status=2）

【一键收货（快捷路径）】
ArrivalNoticeController.oneKeyReceive()
  → ArrivalNoticeServiceImpl.oneKeyReceive()
  → InventoryInServiceImpl.processOneClickArrival()
      → InventoryBaseService.createInventory()  [直接在目标库位入账]
      → InventoryBaseService.createInventoryTransDetail(ARRIVAL_IN)
```

### 6.2 出库核心链路

```
【接单】
OutboundOrderController.add()
  → OutboundOrderServiceImpl.addOrder()
  → wms_outbound_order 表（新建）

【发放拣货】
OutboundOrderController.execute()
  → PickingOrderServiceImpl.createByOutbound()
  → inventoryService.getListByStrategy()  [查可用库存]
  → inventoryOutService.plannedPicking()
      → InventoryBaseService.updateInventoryByLock(OUTBOUNT)  [锁定库存]
  → wms_picking_detail + wms_picking_order（生成拣货）
  → wms_outbound_order（order_status=3/2）

【发货】
PackingController.ship()
  → PackingServiceImpl.ship()
  → inventoryOutService.shipping()
      → InventoryBaseService.updateInventoryByUnlock()  [解锁]
      → InventoryBaseService.decrCurrInventory()  [扣减库存]
      → InventoryBaseService.createInventoryTransDetail(SHIPPING_OUT)
  → wms_outbound_order（order_status=8/9）
  → erpService.request(OUT_FINISH)  [回传ERP]
```

## 七、常见问题 Q&A

| 问题 | 快速答案 |
|------|---------|
| 验收后为什么 available_quantity 还是 0？ | 货在收货库位，被上架锁定（lockType=1），等 PDA 上架到目标库位后才可用 |
| 出库分配后库存怎么看被锁了多少？ | 查 wms_inventory_lock 表，lockType=2 是出库锁定 |
| 单号格式是什么？ | bizCode + yyyyMMdd + 6位序号，如 A20260430000001 |
| 报损和报溢的流水类型各是多少？ | 报损=14(LOSS_OUT)，报溢=15(SURPLUS_IN) |
| 为什么操作库存需要行锁？ | 防止并发：多请求同时修改同一库存行会导致数量计算错误 |
| 冻结和锁定的区别？ | 锁定=系统自动/临时；冻结=人工/持久，需审批解冻 |
| 发货后 ERP 怎么知道？ | erpService.request(EErpApi.OUT_FINISH, ...) 异步回传 |
| 取样后零件上架为什么单独建上架单？ | 取样后数量不完整，零件需要单独上架到特定位置 |
| 盘点单本身会改库存吗？ | 不会！盘点只记录差异，生成损溢单并审批后才改库存 |
